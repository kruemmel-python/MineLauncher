      <section id="overview">
        <h2>Overview</h2>
        <p>MineLauncher RPG is a full MMORPG framework for PaperMC servers. It offers classes, skills, quests, dungeons, guilds, trading, PvP, permissions, and a build system with native Sponge schematics.</p>
        <div class="grid-2">
          <div class="callout">
            <strong>Highlight:</strong> Native <code>.schem</code> engine without WorldEdit, including BlockEntities, Entities, and Undo functionality.
          </div>
          <div class="callout">
            <strong>Note:</strong> PostgreSQL is mandatory for persistent player data.
          </div>
        </div>
      </section>

      <section id="requirements">
        <h2>Prerequisites</h2>
        <ul>
          <li><span class="badge">Server</span> PaperMC 1.20.4+</li>
          <li><span class="badge">Java</span> JDK 17+</li>
          <li><span class="badge">Database</span> PostgreSQL (Required)</li>
        </ul>
      </section>

      <section id="installation">
        <h2>Installation & Database</h2>
        <ol>
          <li>Place the plugin <code>rpg-plugin-1.0.0.jar</code> in the <code>plugins/</code> folder.</li>
          <li>Start the server once and then stop it to generate configs.</li>
          <li>Open <code>plugins/RPGPlugin/config.yml</code> and set your PostgreSQL credentials.</li>
        </ol>
        <pre><code>database:
  host: localhost
  port: 5432
  name: rpg_db
  user: your_user
  password: your_password
  poolSize: 10
</code></pre>
        <p>The plugin automatically creates the tables upon startup.</p>
      </section>

      <section id="core-config">
        <h2>Core Configuration</h2>
        <h3>Important Files</h3>
        <ul>
          <li><code>config.yml</code> – Global settings (Database, Dungeon, Schematics, Permissions)</li>
          <li><code>skills.yml</code> – Skills</li>
          <li><code>classes.yml</code> – Classes</li>
          <li><code>mobs.yml</code> – Custom Mobs</li>
          <li><code>loot.yml</code> – Loot Tables</li>
          <li><code>shops.yml</code> – Shops</li>
          <li><code>buildings.yml</code> – Building definitions</li>
        </ul>
      </section>

      <section id="launcher-cli">
        <h2>Launcher CLI (Outside the Game)</h2>
        <p>The C# launcher supports a CLI start for the client:</p>

# Example
dotnet run --project C_launcher -- play Alex
</code></pre>

      <section id="buildings">
        <h2>Buildings & Schematics</h2>
        <h3>Providing Schematics</h3>
        <p>Place <code>.schem</code> files under <code>plugins/RPGPlugin/schematics/</code>.</p>
        <h3>Defining Building</h3>
        <pre><code>buildings:
  cottage:
    name: "Small House"
    category: RESIDENTIAL
    schematic: "cottage_ground.schem"
    floorSchematic: "cottage_floor.schem"
    minFloors: 1
    maxFloors: 2
    floorHeight: 5
    basement:
      schematic: "cottage_basement.schem"
      depth: 4
</code></pre>
        <h3>Placing Single Schematic</h3>
        <ol>
          <li>Open Admin Menu: <code>/rpgadmin</code></li>
          <li>Build Manager → Single Schematic</li>
          <li>Enter filename (e.g., <code>house.schem</code>)</li>
          <li>Right-click to place</li>
        </ol>
        <h3>Undo</h3>
        <p><code>/rpgadmin build undo</code> reverts the last placement per admin.</p>
      </section>
