MineLauncher/MainForm.cs

The MainForm class serves as the primary graphical user interface for the MineLauncher application. Built using Windows Forms, it provides a dashboard for managing Minecraft server configurations, downloading server binaries, and launching both the server and the client.

Class: MainForm

Description: A sealed class inheriting from Form that coordinates the interaction between the user and various backend services (Config, Download, Server, and Launcher services).

Constructor

  • MainForm(string basePath, ConfigService configService, DownloadService downloadService, ServerService serverService, LauncherService launcherService, ClientLauncherService clientLauncherService): Initializes the UI components, styles the buttons, sets up event handlers, and loads the initial configuration.

Private Methods

  • StyleButton(Button button)

    Applies a consistent dark-themed visual style to the UI buttons, including custom colors, fonts, and border properties.

  • InitConfigAsync()

    Saves the current configuration object to the disk and refreshes the editor view.

  • OpenConfig()

    Opens the configuration file (launcher_config.json) in the system's default text editor.

  • ShowConfig()

    Reloads the configuration from the file and displays it in the internal text editor.

  • LoadConfigToEditor()

    Synchronizes the _config object with the latest data from the ConfigService and updates the text box.

  • SaveConfigFromEditor()

    Attempts to parse the text from the internal editor and save it back to the configuration file. Includes error handling for invalid formats.

  • DownloadServerAsync()

    Handles the logic for downloading the server JAR. It determines the correct URL based on the server type (Paper or Vanilla) and version specified in the config, then ensures necessary files like eula.txt and server.properties are created.

  • LaunchServerAsync()

    Prepares the environment and executes the server JAR. If the JAR is missing, it triggers the download process automatically before launching.

  • LaunchClientAsync()

    Invokes the ClientLauncherService to start the Minecraft client using the username and version defined in the configuration.

  • RunActionAsync(Func<Task> action)

    A wrapper for asynchronous UI actions. It disables buttons during execution to prevent race conditions and logs any exceptions to the status box.

  • SetButtonsEnabled(bool enabled)

    Toggles the Enabled state of all primary action buttons.

  • AppendStatus(string message)

    Appends a timestamped log message to the _statusBox (the console-like output area in the UI).

UI Components

  • _statusBox: A read-only TextBox used for logging application activity.
  • _configEditor: A multi-line TextBox allowing users to manually edit the JSON configuration.
  • buttonPanel: A FlowLayoutPanel containing the action buttons (Init, Open, Download, Launch, etc.).
  • contentPanel: A SplitContainer that divides the view between the status log and the config editor.

MineLauncher/Models/LauncherConfig.cs

This file defines the data structures used for configuring the Minecraft launcher, server environments, and Java runtime settings. These classes are typically used for serialization and deserialization of configuration files (e.g., JSON or YAML).

LauncherConfig (Class)

The root configuration class that aggregates all settings for the application.

  • InstallRoot (string): The base directory where servers and game files are stored. Defaults to "servers".
  • Server (ServerConfig): Configuration object for the server instance.
  • Java (JavaConfig): Configuration object for the Java Runtime Environment.
  • Game (GameConfig): Configuration object for the client-side game settings.

ServerConfig (Class)

Contains settings specific to the Minecraft server instance being managed.

  • Name (string): The display name or folder name for the server.
  • Type (string): The server software type. Supported values include vanilla, paper, bukkit, and craftbukkit.
  • Version (string): The Minecraft version (e.g., "1.20.4").
  • PaperBuild (int?): Optional specific build number for PaperMC servers.
  • OnlineMode (bool): Whether the server validates players with Mojang/Microsoft accounts.
  • Port (int): The network port the server listens on. Defaults to 25565.
  • AcceptEula (bool): Indicates if the user has accepted the Minecraft End User License Agreement.
  • JarFileName (string): The name of the executable JAR file for the server.

JavaConfig (Class)

Defines the parameters for the Java Virtual Machine (JVM) used to run the server or client.

  • Path (string): The path to the Java executable. Defaults to "java" (system path).
  • MinMemory (string): The initial heap size (Xms), e.g., "1G".
  • MaxMemory (string): The maximum heap size (Xmx), e.g., "2G".

GameConfig (Class)

Settings related to the Minecraft client launcher functionality.

  • ClientVersion (string): The version of the Minecraft client to launch.
  • GameDirectory (string): The specific folder where client assets and data are stored.
  • MaxMemoryMb (int): Maximum memory allocated to the client in Megabytes.
  • OfflineMode (bool): If true, the client launches without authenticating against official servers.
  • OfflineUsername (string): The username used when OfflineMode is enabled.

MineLauncher/Program.cs

The Program.cs file serves as the main entry point for the CLauncher application. It handles the initial application configuration, dependency injection for core services, and determines whether to launch the graphical user interface (GUI) or execute specific command-line instructions.

Class: Program

Type: internal static class

Purpose: This class contains the static Main method which initializes the application environment and manages the lifecycle of the launcher.

Method: Main()

Attributes: [STAThread]

Purpose: The primary entry point for the application. It initializes the Windows Forms configuration, sets up file paths for configuration, and instantiates the necessary services for the application to function.

Key Responsibilities:

  • Initialization: Calls ApplicationConfiguration.Initialize() to prepare the UI environment.
  • Service Orchestration: Instantiates core services including:
    • ConfigService: Manages the launcher-config.json file.
    • DownloadService: Handles HTTP requests for updates and assets.
    • ServerService: Manages server-related logic.
    • LauncherService: General launcher operations.
    • ClientLauncherService: Specifically handles the execution of the game client.
  • CLI Argument Handling:

    Checks for command-line arguments. If the play command is detected, it bypasses the GUI to launch the game client directly using either a provided username or the default offline username from the configuration.

  • GUI Launch: If no specific command-line arguments are processed, it starts the standard Windows Forms application by running MainForm with the injected services.

Workflow Logic:

if (command == "play") {
  // Launch client asynchronously and wait for completion
  clientLauncherService.LaunchClientAsync(...).GetAwaiter().GetResult();
} else {
  // Run the MainForm GUI
  Application.Run(new MainForm(...));
}

MineLauncher/README.md

This file serves as the primary documentation and user guide for the C_launcher project and the associated MineLauncherRPG plugin. It provides setup instructions, configuration details, and a comprehensive list of in-game commands for players and administrators.

Project Overview

The project consists of a C#-based launcher with a GUI for managing Minecraft servers (Vanilla, Paper, Bukkit, CraftBukkit) and an integrated client launcher for offline mode. It also includes a complex RPG plugin for Minecraft.

Launcher Components

  • GUI Launcher: Handles server downloads, configuration, and execution.
  • Client Launcher: Downloads Minecraft assets and launches the game in offline mode.
  • CLI Interface: Allows starting the client via command line using dotnet run --project C_launcher -- play [Name].

Configuration (launcher-config.json)

The launcher is configured via a JSON file with the following key sections:

    • Return Value:
      • BehaviorStatus.SUCCESS: Returned immediately if any child node returns success.
      • BehaviorStatus.RUNNING: Returned immediately if any child node returns running.

Logic Flow

  1. Iterates through the list of children().
  2. Calls tick(context) on the current child.

  4. If the child returns FAILURE, the Selector continues to the next child in the sequence.
  5. If the loop completes without any child returning success or running, the node returns FAILURE.
